2) Replace your “entries area” with one draggable list

Below is a minimal but working rewrite of just the entries block in your screen. Everything above (date header, summary) stays the same. The important parts are:

buildVirtualDay() flattens headers + items into a single array.

renderRow() draws either a header or a FoodEntryCard.

onDragEnd() re-derives each item’s meal_name from the nearest header above and updates local state + Supabase.

Tip: skim the comments; they map exactly to the behaviors you described (placeholder space, cross-section moves, etc.).

// ADD these imports at the top:
import DraggableFlatList, {
  ScaleDecorator,
} from "react-native-draggable-flatlist";
import Animated, { Layout } from "react-native-reanimated";
import { useSafeAreaInsets } from "react-native-safe-area-context";

// … keep your other imports

type VirtualRow =
  | { kind: "header"; key: string; title: string; icon: string; mealType: number }
  | { kind: "item"; id: string; mealType: number; entry: FoodEntry };

export default function DiaryPage() {
  // ... keep your existing state/hooks
  const insets = useSafeAreaInsets();

  // --- Helpers to (a) partition and (b) flatten to 1 list
  const getEntriesForMeal = React.useCallback(
    (mealType: number) =>
      foodEntries
        .filter((e) => e.meal_name === mealType)
        .sort(
          (a, b) =>
            new Date(a.created_at).getTime() - new Date(b.created_at).getTime()
        ),
    [foodEntries]
  );

  const MEAL_SECTIONS = React.useMemo(
    () => [
      { key: "breakfast", title: "Breakfast", icon: "sunny-outline", mealType: 1 },
      { key: "lunch", title: "Lunch", icon: "fast-food-outline", mealType: 2 },
      { key: "dinner", title: "Dinner", icon: "restaurant-outline", mealType: 3 },
      { key: "snack", title: "Snacks", icon: "ice-cream-outline", mealType: 4 },
      { key: "uncategorized", title: "Other", icon: "ellipsis-horizontal-outline", mealType: 0 },
    ],
    []
  );

  const buildVirtualDay = React.useCallback((): VirtualRow[] => {
    const rows: VirtualRow[] = [];
    for (const s of MEAL_SECTIONS) {
      rows.push({
        kind: "header",
        key: s.key,
        title: s.title,
        icon: s.icon,
        mealType: s.mealType,
      });
      const items = getEntriesForMeal(s.mealType);
      for (const entry of items) {
        rows.push({
          kind: "item",
          id: entry.id,
          mealType: s.mealType,
          entry,
        });
      }
    }
    return rows;
  }, [MEAL_SECTIONS, getEntriesForMeal]);

  const [virtualRows, setVirtualRows] = React.useState<VirtualRow[]>([]);

  // rebuild whenever entries change
  React.useEffect(() => {
    setVirtualRows(buildVirtualDay());
  }, [buildVirtualDay]);

  // --- Persist to Supabase (optimistic)
  const persistMoves = React.useCallback(
    async (updated: VirtualRow[]) => {
      // Build batched updates: for each item row, compute meal + order index
      const updates: { id: string; meal_name: number; order_index: number }[] = [];
      const orderByMeal: Record<number, number> = {}; // per-meal running index

      // walk top to bottom; track current meal (last header seen)
      let currentMeal = 0;
      for (const row of updated) {
        if (row.kind === "header") {
          currentMeal = row.mealType;
          orderByMeal[currentMeal] = 0;
        } else {
          updates.push({
            id: row.id,
            meal_name: currentMeal,
            order_index: (orderByMeal[currentMeal] ?? 0),
          });
          orderByMeal[currentMeal] = (orderByMeal[currentMeal] ?? 0) + 1;
        }
      }

      // optimistically update local state first
      setFoodEntries((prev) => {
        const map = new Map(prev.map((e) => [e.id, e]));
        for (const u of updates) {
          const e = map.get(u.id);
          if (e) {
            map.set(u.id, {
              ...e,
              meal_name: u.meal_name,
              // optional: set created_at to keep visual order consistent
              created_at: e.created_at,
            });
          }
        }
        return Array.from(map.values());
      });

      // persist in background (best-effort); ignore order_index in DB if you sort by created_at
      try {
        // Example: update meal_name (and optionally an order column if you add one)
        const payload = updates.map(({ id, meal_name }) => ({ id, meal_name }));
        const { error } = await supabase.from("food_entry").upsert(payload);
        if (error) {
          console.error("Supabase persist error:", error);
          // Optionally refetch on failure
          // await fetchFoodEntries();
        }
      } catch (err) {
        console.error("Persist failed:", err);
      }
    },
    [setFoodEntries, supabase]
  );

  // --- Drag handlers
  const onDragEnd = React.useCallback(
    async ({ data }: { data: VirtualRow[] }) => {
      // 1) Keep the new virtual order
      setVirtualRows(data);
      // 2) Recompute each item’s meal based on nearest header above, then persist
      await persistMoves(data);
    },
    [persistMoves]
  );

  // --- Row renderer
  const renderRow = React.useCallback(
    ({ item, drag, isActive }: any) => {
      if (item.kind === "header") {
        // Sticky-like header row that’s NOT draggable
        return (
          <Animated.View
            layout={Layout.springify()}
            className="bg-gray-800 rounded-xl px-4 py-3 mt-2"
          >
            <View className="flex-row items-center justify-between">
              <View className="flex-row items-center">
                <Ionicons
                  name={item.icon as any}
                  size={18}
                  color={"#CFB991"}
                />
                <Text className="text-sm font-sora-semibold ml-2 text-white">
                  {item.title}
                </Text>
              </View>
            </View>
          </Animated.View>
        );
      }

      // Draggable food item
      return (
        <ScaleDecorator>
          <Animated.View
            layout={Layout.springify()}
            className="mt-2 ml-2"
          >
            <FoodEntryCard
              entry={item.entry}
              onRemove={handleRemoveEntry}
              // make the whole card draggable by long-pressing
              onLongPress={drag}
              isActive={isActive} // you can style the card slightly when active
            />
          </Animated.View>
        </ScaleDecorator>
      );
    },
    [handleRemoveEntry]
  );

  // --- Key extractor for virtual rows
  const keyExtractor = React.useCallback((row: VirtualRow, index: number) => {
    return row.kind === "header" ? `h-${row.key}` : `i-${row.id}`;
  }, []);

  // --- Your existing effect still fetches entries:
  React.useEffect(() => {
    fetchFoodEntries();
  }, [user, selectedDate]);

  return (
    <BackgroundTemplate>
      {/* keep your date header and summary exactly as-is */}

      <View className="flex-1 px-6 pt-14 pb-4">
        {/* Date header + DailyProgress + Add button ... unchanged */}

        {/* Draggable list replaces the per-section maps */}
        {loading ? (
          <View className="bg-gray-800 rounded-xl p-6 items-center">
            <Text className="text-white text-base font-sora">Loading...</Text>
          </View>
        ) : (
          <DraggableFlatList
            containerStyle={{ paddingBottom: insets.bottom + 16 }}
            data={virtualRows}
            keyExtractor={keyExtractor}
            renderItem={renderRow}
            onDragEnd={onDragEnd}
            activationDistance={12} // easier to pick up without accidental drags
            dragItemOverflow // allow overlay to float
            autoscrollSpeed={300}
            autoscrollThreshold={80}
            // Optional: prevent headers from being dragged
            onDragBegin={(index) => {
              if (virtualRows[index]?.kind === "header") {
                // cancel if user grabbed a header
                // (library doesn’t expose cancel; instead require long press on item only)
              }
            }}
          />
        )}
      </View>
    </BackgroundTemplate>
  );
}

Why this feels like Cronometer

The library renders a drag overlay (the “ghost”), and the list leaves a placeholder so rows shift around you.

Because headers live in the same list, hovering across headers naturally changes where the placeholder sits. On drop, we compute the target meal from the last header above the item and update meal_name.

Auto-scroll comes for free when you hover near the edges (autoscrollThreshold, autoscrollSpeed).

3) (Optional) Expand/collapse behavior during drag

You can keep your expandedMeals state for tap-to-collapse when not dragging. During a drag, most apps temporarily keep all sections open so space is predictable. A simple rule:

When a drag starts, expand all sections.

When it ends, restore the user’s previous expanded/collapsed state.

You can toggle a dragging boolean via onDragBegin/onDragEnd and conditionally render collapsed headers as expanded.
